# 📜Regras do Projeto


## 1. Organizão Geral
* projeto deve ser organizado em pastas.
* cada disciplina é responsavel por sua área.
<br>

## 2. Como usar o GitHub
* Ninguém deve trabalhar diretamente na branch `main` .
* Cada equipe deve utilizar sua própria branch.
* Todo código **deve passar por revisão antes de ir para branch** `main`

## Commits
* **Sempre colocar descrição com objetivo claro** antes de fazer um commit.
<br>


## 3. Responsabilidade
* Cada membro é responsável por sua entrega.
* **Falta de comunicação impacta o grupo inteiro** .

## comunicação
* Todos os membros devem responder mensagens importantes.
* Relatar problemas ou difilcudades o quanto antes.

## Prazos
* Entregaras devem ser feitas antes do prazo combinado.
* Caso não consiga, avisar o quanto antes.

## Qualidade
* Não subir código incompleto sem aviso.
* Manter organização e clareza.

## Trabalho em Equipe
* Ajudar outros membros quando possível.
* O projeto é coletivo não individual.
